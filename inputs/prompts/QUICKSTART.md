# 빠른 시작 프롬프트

> 복사해서 Claude에 붙여넣기

---

## 1️⃣ 원클릭 전체 생성 (권장)

```
원클릭 실행
```

---

## 2️⃣ 단계별 실행

### Phase 0: 스캔
```
inputs 폴더 스캔해서 INDEX.md 만들어줘
```

### Phase 1: 분석
```
inputs 폴더의 모든 파일을 읽고 결정사항과 미확정 항목을 분리해줘
```

### Phase 2: 정의
```
분석 결과를 기반으로 기능정의서, 화면정의서, AC_EdgeCase, 용어정의서를 생성해줘
```

### Phase 3: WBS
```
기능정의서를 기반으로 WBS와 Gantt 차트를 생성해줘
```

### Phase 4: 검증
```
outputs 폴더의 산출물들 일관성 체크하고 핸드오프 준비 상태 확인해줘
```

---

## 3️⃣ 개별 산출물 생성

### 기능정의서만
```
inputs 자료 기반으로 기능정의서만 만들어줘
```

### 화면정의서만
```
기능정의서 보고 화면정의서만 작성해줘
```

### WBS만
```
기능정의서 기반으로 WBS만 만들어줘
```

### 견적서 추가
```
원클릭 실행하고 견적서도 같이 만들어줘
M/D 단가는 76만원으로
```

---

## 4️⃣ 수정/보완

### 특정 섹션 수정
```
기능정의서의 3.2절 '주문 처리' 부분을 더 상세하게 수정해줘
```

### 미확정 항목 해결
```
미확정 항목 중 '결제 방식'을 '카드+현금 복합결제'로 확정해줘
```

### 일관성 검토
```
기능정의서와 화면정의서 사이에 불일치 있는지 확인해줘
```

---

## 5️⃣ 특수 상황

### 대용량 파일 처리
```
inputs/회의록/대용량_회의록.txt 파일을 분할해서 읽고 분석해줘
```

### 이미지 분석 포함
```
inputs/참고자료/ 폴더의 스크린샷들도 분석해서 화면정의서에 반영해줘
```

### 기존 산출물 업데이트
```
새로 추가된 요구사항을 반영해서 기능정의서를 업데이트해줘
```

---

## 팁

- **처음 사용**: `"원클릭 실행"` 으로 시작
- **자료가 많을수록** 산출물 품질 향상
- **미확정 항목**은 나중에 개별 수정 가능
- **검토 후 수정**은 해당 섹션만 지정해서 요청
