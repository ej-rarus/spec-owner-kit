# TC 시트 템플릿

> 조건 매트릭스 기반 테스트케이스 시트 구조

---

## 시트 구성

| 시트명 | 용도 |
|--------|------|
| 변수정의 | 드롭다운 값 목록 |
| MO_테스트케이스 | 모바일 TC (MO-001 ~) |
| PC_테스트케이스 | PC TC (PC-001 ~) |
| 결함현황 | PASS/FAIL/N/T 자동 집계 |

---

## 컬럼 구조

### 식별 (노란색)
| 컬럼명 | 입력방식 | 설명 |
|--------|----------|------|
| 결함ID | 텍스트 | FAIL 시 부여 |
| TC_ID | 자동채번 | MO-001, PC-001 |

### 조건 (녹색)
| 컬럼명 | 입력방식 | 기본값 |
|--------|----------|--------|
| 우선순위 | 드롭다운 | Critical/High/Medium/Low |
| 기능영역 | 드롭다운 | 프로젝트별 정의 |
| 페이지 | 텍스트 | - |
| 로그인상태 | 드롭다운 | 로그인/비로그인 |
| 회원유형 | 드롭다운 | 신규/기존/비회원 |
| (도메인 특화) | 드롭다운 | 프로젝트별 추가 |

### 테스트 (파란색)
| 컬럼명 | 입력방식 | 기본값 |
|--------|----------|--------|
| 동작 | 드롭다운 | 진입/조회/추가/삭제/수정/결제... |
| 기대결과 | 텍스트 | - |

### 수행결과 (분홍색)
| 컬럼명 | 입력방식 | 기본값 |
|--------|----------|--------|
| 담당자 | 텍스트 | 직접입력 |
| QA 결과 | 드롭다운 | PASS/FAIL/N/T |
| 등록일 | 날짜 | yyyy-mm-dd |
| 실제결과 | 텍스트 | FAIL 시 작성 |
| 조치권고(수정사항) | 텍스트 | - |
| 결함심각도 | 드롭다운 | Critical/Major/Minor/Trivial |
| 결함유형 | 드롭다운 | 8종 |

### 관리 (회색)
| 컬럼명 | 입력방식 |
|--------|----------|
| 비고 | 텍스트 |

## 결함심각도 정의

| 값 | 설명 |
|----|------|
| Critical | 서비스 불가, 데이터 손실, 결제 오류 |
| Major | 주요 기능 동작 안함, 우회 불가 |
| Minor | 기능 동작하나 불편, 우회 가능 |
| Trivial | UI 깨짐, 오타, 미미한 이슈 |

---

## 결함유형 정의

| 값 | 설명 |
|----|------|
| 기능구현 누락 | 요구사항 대비 미구현된 기능 |
| Validation 미처리 | 입력값 검증, 예외처리 누락 |
| UI 결함 | 레이아웃, 디자인, 반응형 깨짐 |
| 표기 오류 | 오타, 잘못된 문구, 번역 오류 |
| 사용성 개선 | UX 불편, 개선 권고사항 |
| 응답속도 지연 | 로딩 느림, 타임아웃 |
| 연동 오류 | API, 외부 시스템 연동 실패 |
| 데이터처리 오류 | 잘못된 값 표시, 저장/조회 오류 |
