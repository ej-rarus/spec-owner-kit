# 원클릭 산출물 생성 가이드

> **하나의 명령으로 5개 산출물 자동 생성**

---

## 사용법

### 1. 자료 준비
```
inputs/
├── 회의록/     ← 고객 미팅 기록 (.md, .docx, .txt)
├── 요구사항/   ← 고객 요청사항
└── 참고자료/   ← 벤치마킹, 스크린샷 (.png, .jpg)
```

### 2. 프로젝트 정보 설정
아래 `ONECLICK.md` 하단의 **프로젝트 설정** 섹션 수정

### 3. 원클릭 실행
```
"원클릭 실행"
```

또는 상세 버전:
```
"inputs 폴더의 모든 자료를 기반으로 기능정의서, 화면정의서, WBS, AC_EdgeCase, 용어정의서를 한번에 생성해줘"
```

```yaml
  - 화면정의서      # 필수

  - 견적서         # 선택 (MD_RATE 필요)
```

## 기본 결정 사항 (Default Decisions)

> 미확정 항목 발생 시 기본 적용되는 결정들
> 특수한 경우 아래 값을 수정하거나, 생성 후 개별 수정

```yaml
# 스코프 판단 기준
SCOPE_DEFAULTS:
  unknown_feature: "OUT"     # 명시 안된 기능 → 제외
  nice_to_have: "OUT"        # 있으면 좋은 기능 → 제외
  performance_optimization: "IN"  # 성능 최적화 → 포함

# 우선순위 기준
PRIORITY_DEFAULTS:
  core_business: "P0"        # 핵심 비즈니스 로직
  user_facing: "P1"          # 사용자 화면 기능
  admin_feature: "P2"        # 관리자 기능
  reporting: "P2"            # 리포팅/통계

# 기술 결정
TECH_DEFAULTS:
  frontend: "Shopify Liquid + Alpine.js"
  middleware: "Node.js + Express"
  database: "PostgreSQL"
  cache: "Redis"
  hosting: "AWS"

# WBS 역할 배분
ROLE_DEFAULTS:
  ui_composer: 0.25          # UI 구성 25%
  ui_integrator: 0.25        # UI 연동 25%
  integrator: 0.35           # 백엔드 35%
  verifier: 0.15             # QA 15%
```

## AI 지시사항 (원클릭 모드)

> Claude가 "원클릭 실행" 명령을 받으면 아래를 자동 수행

### 원클릭 실행 조건
- `"원클릭"`, `"원클릭 실행"`, `"전체 생성"`, `"한번에 만들어줘"` 키워드 감지
- 또는 `"inputs 기반으로 ... 한번에"` 형태의 요청

### 자동 수행 단계

```markdown
## STEP 1: 스캔
1. `ls -laR inputs/` 실행하여 모든 파일 확인
2. 샘플 파일 (_샘플_, _samples/) 제외
3. inputs/INDEX.md 생성/업데이트
4. (확인 건너뛰기 - 원클릭 모드)

## STEP 2: 분석
1. inputs/회의록/ 의 모든 파일 읽기
2. inputs/요구사항/ 의 모든 파일 읽기
3. inputs/참고자료/ 이미지 분석 (있는 경우)
4. 결정사항/미확정사항 분류
5. 미확정 → ONECLICK.md의 Default Decisions 적용
6. (확인 건너뛰기 - 원클릭 모드)

## STEP 3: 생성
1. inputs/templates/ 참조하여 산출물 형식 확인
2. 기능정의서.md 생성 → outputs/
3. 화면정의서.md 생성 → outputs/
4. AC_EdgeCase.md 생성 → outputs/
5. 용어정의서.md 생성 → outputs/
6. WBS.md 생성 → outputs/
7. (견적서 - 설정 시) 견적서.html 생성 → outputs/

## STEP 4: 검증
1. 산출물 간 ID/용어 일관성 확인
2. 누락된 기능/화면 크로스체크
3. 생성 완료 리포트 출력
```

### 완료 리포트 형식
```markdown
## ✅ 원클릭 생성 완료

### 입력 자료
- 회의록: 3개 파일
- 요구사항: 2개 파일
- 참고자료: 5개 이미지

### 생성된 산출물
| 산출물 | 상태 | 파일 |
|--------|------|------|
| 기능정의서 | ✅ | outputs/기능정의서.md |
| 화면정의서 | ✅ | outputs/화면정의서.md |
| WBS | ✅ | outputs/WBS.md |
| AC_EdgeCase | ✅ | outputs/AC_EdgeCase.md |
| 용어정의서 | ✅ | outputs/용어정의서.md |

### 주의사항
- 미확정 항목 3개 → 기본값 적용됨 (검토 필요)
- [미확정 항목 목록...]

### 다음 단계
1. `outputs/` 폴더의 산출물 검토
2. 미확정 항목 확정 필요 시 개별 수정
3. 팀 핸드오프 진행
```

## 커스텀 명령어

| 명령어 | 설명 |
|--------|------|
| `"원클릭 실행"` | 전체 산출물 한번에 생성 |
| `"원클릭 + 견적서"` | 전체 산출물 + 견적서 포함 |

## FAQ

### Q: 원클릭 중 에러가 나면?
에러 발생 시 해당 단계에서 중단하고 상태 보고합니다.
`"이어서 실행"` 명령으로 재개 가능합니다.

### Q: 미확정 항목이 많으면?
Default Decisions가 자동 적용됩니다.
중요한 결정은 생성 후 `outputs/` 산출물에서 수정하세요.

### Q: 기존 산출물을 덮어쓰나요?
네. 기존 파일이 있으면 덮어씁니다.
백업이 필요하면 먼저 복사해두세요.

### Q: 대용량 파일이 있으면?
50KB 이상 파일은 자동으로 분할 읽기합니다.
처리 시간이 조금 더 걸릴 수 있습니다.
